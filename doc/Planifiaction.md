| Semaine | Objectifs principaux                                    | Détails techniques / tâches                                                                                                                                                                                                             | Livrables                                   |
| ------- | ------------------------------------------------------- | --------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- | ------------------------------------------- |
| **S1**  | **Analyse du projet et conception de l’interface (UI)** | - Maquette de l’interface (Figma)<br> - Création du projet Windows Forms<br>- Maquette de l’interface (boutons : Ajouter, Envoyer, Télécharger, etc.)<br>- Définition des classes principales (FileHandler, Peer, ConnectionManager)    | Interface de base fonctionnelle             |
| **S2**  | **Architecture réseau et structure P2P**                | - Création de la classe `Peer` pour gérer la connexion entre utilisateurs<br>- Sockets TCP/UDP de base pour échanger les métadonnées<br>- Tests de communication entre deux pairs locaux                                                | Prototype de communication P2P              |
| **S3**  | **Implémentation du chunking (partage de fichiers)**    | - Découpage des fichiers en chunks (byte[])<br>- Envoi/réception de chunks via sockets<br>- Sauvegarde temporaire des blocs reçus                                                                                                       | Transfert simple d’un fichier en chunks     |
| **S4**  | **Reconstruction et intégrité**                         | - Reconstitution du fichier complet<br>- Vérification d’intégrité (hash MD5/SHA1)<br>- Interface d’affichage de l’état du téléchargement                                                                                                | Fichier complet reconstitué avec validation |
| **S5**  | **Gestion du timeout et des erreurs réseau**            | - Mise en place d’un timer pour surveiller les réponses<br>- Relance automatique des requêtes perdues<br>- Logs d’erreurs affichés dans la console/label                                                                                | Système de retransmission automatique       |
| **S6**  | **Implémentation du sdelay (Smart Delay)**              | - Ajout d’un délai dynamique entre les envois de chunks selon :<br> → la charge réseau<br> → le nombre de peers actifs<br> → le taux de réussite des transmissions<br>- Calcul du _sdelay_ avec adaptation automatique (ex : 50–300 ms) | Transfert fluide sans congestion            |
| **S7**  | **Optimisation et interface avancée**                   | - Indication visuelle du débit et du délai<br>- Réglage manuel du _sdelay_ via un slider                                                                                                                                                | Interface finalisée et dynamique            |
| **S8**  | **Tests finaux et documentation**                       | - Tests multi-pairs sur le réseau local<br>- Correction des bugs<br>- Rédaction du README et rapport technique                                                                                                                          | Application complète et documentée          |
